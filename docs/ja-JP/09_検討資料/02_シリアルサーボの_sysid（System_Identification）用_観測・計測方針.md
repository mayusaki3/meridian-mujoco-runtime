# シリアルサーボの sysid（System Identification）用 観測・計測方針

# 概要

シリアルサーボ（例：ICS3.5/3.6）の system identification（sysid）では、

- サーボへ送った指示
- サーボ内部状態
- 外部センサによる実物理挙動
- 外部負荷
- 電源状態

を同時刻基準で観測・記録する必要がある。

特に重要なのは、

```text
「何を指示したか」
```

を観測対象として扱うことである。

sysid は、

```text
入力 → 内部制御 → 出力
```

の関係を推定するため、

- 指示値
- 指示タイミング
- 指示更新周期

は極めて重要な観測データとなる。

---

# sysid を実機で決定する場合の基本方針

## 基本構造

実機sysidでは、

```text
実機へ入力を与える
  ↓
実機挙動を観測する
  ↓
シミュレーションとの差を計算する
  ↓
パラメータを更新する
```

を繰り返す。

---

# sysid が推定するもの

## 代表例

- 剛性（kp）
- ダンピング（kd）
- 摩擦
- 最大速度
- 最大トルク
- 電流制限
- 応答遅れ
- デッドバンド
- 初動補正（パンチ）
- 温度依存特性

---

# なぜ実機観測が必要か

例えば：

```text
stretch = 60
```

という設定値だけでは、

```text
どの程度の保持力になるか
```

は分からない。

必要なのは：

```text
設定値
  ↓
実際の挙動
```

の関係である。

そのため、

- 実角度
- 実速度
- 実電流
- 実負荷

を観測する必要がある。

---

# 基本方針

## sysidで観測すべき対象

以下をすべて観測対象とする。

| 分類 | 内容 |
|---|---|
| 指示 | 目標角、速度指示、設定変更 |
| 内部状態 | 現在角、電流、温度、保護状態 |
| 外部物理状態 | 実角度、実速度、実加速度 |
| 負荷 | 外力、トルク、拘束状態 |
| 電源 | 電圧、電流 |
| 環境 | 温度、姿勢、重力方向 |

---

# なぜ「指示」が必要か

## sysidは入力と出力の関係を推定する

例えば：

```text
時刻 0ms:
  target = 0°

時刻 100ms:
  target = 90°
```

という入力に対し、

```text
実角度:
  0° → 20° → 45° → 70° → 88°
```

のような応答が発生する。

sysid は、

```text
入力:
  target

出力:
  actual angle
```

の差から、

- 応答遅れ
- 剛性
- 摩擦
- 最大速度
- 最大トルク

などを推定する。

そのため、

```text
何をいつ指示したか
```

は必須情報となる。

# 推奨観測構成

## 全体構成

```text
PC / 制御マイコン
  ├─ 指示ログ
  │
  ├─ ICSシリアル通信
  │
  ├─ 外部角度センサ
  │
  ├─ 電流・電圧センサ
  │
  ├─ 負荷センサ
  │
  └─ 共通タイムスタンプロガー
```

---

# 観測対象詳細

# 1. 指示ログ

## 目的

```text
サーボへ与えた入力を記録する
```

---

## 必須項目

| 項目 | 内容 |
|---|---|
| timestamp | 指示送信時刻 |
| servo_id | サーボ識別 |
| command_type | 指示種別 |
| target_position | 目標角 |
| target_velocity | 目標速度 |
| target_torque | 目標トルク |
| parameter_write | 設定変更 |
| raw_packet | 生シリアルデータ |

---

## 重要性

例えば：

```text
90°指示
```

だけでは不十分である。

必要なのは：

```text
いつ
どの周期で
どの補間で
何回送ったか
```

である。

---

## 記録推奨

### 生パケット保存

推奨：

```text
送受信シリアル生データを保存する
```

理由：

- デコードミス検証
- 再解析
- 通信遅延解析
- 将来仕様変更対応

が可能になる。

---

# 2. ICS内部状態

## 目的

```text
サーボ内部が認識している状態を取得する
```

---

## 推奨取得項目

| 項目 | 内容 |
|---|---|
| current_position | 現在角 |
| current_value | 電流 |
| temperature | 温度 |
| status_flags | 保護状態 |
| speed_setting | speed |
| stretch_setting | stretch |
| punch_setting | punch |
| deadband_setting | deadband |
| response_setting | response |
| damping_setting | damping |

---

## 注意点

ICS内部値は：

```text
「サーボ内部推定値」
```

であり、

```text
「実物理値」
```

とは限らない。

そのため：

```text
ICS値 = 補助観測
外部センサ = 正観測
```

とするのが望ましい。

---

# 3. 外部角度観測

## 目的

```text
実際の軸角度を観測する
```

---

## 推奨センサ

| センサ | 特徴 |
|---|---|
| 光学式エンコーダ | 高精度 |
| 磁気式エンコーダ | 実装容易 |
| 絶対角エンコーダ | 電源断対応 |

---

## 推奨取得項目

| 項目 | 内容 |
|---|---|
| external_angle | 実角度 |
| external_velocity | 実速度 |
| external_acceleration | 実加速度 |

---

## なぜ必要か

ICS現在値だけでは：

- バックラッシュ
- 軸たわみ
- ギア遅れ
- 通信周期影響

を分離できない。

---

# 4. 電流・電圧観測

## 目的

```text
実際の電力消費・出力状態を観測する
```

---

## 推奨センサ

| センサ | 用途 |
|---|---|
| INA219 | 簡易 |
| INA226 | 高精度 |
| シャント抵抗 + ADC | 高速 |
| 電流プローブ | 高帯域 |

---

## 推奨取得項目

| 項目 | 内容 |
|---|---|
| supply_voltage | 電源電圧 |
| supply_current | 電源電流 |
| power_consumption | 消費電力 |

---

## 観測できるもの

- パンチ
- 電流制限
- 保護動作
- stall
- 温度上昇

---

# 5. 負荷観測

## 目的

```text
負荷条件を定量化する
```

---

## 方法1：既知アーム長 + 重り

```text
τ = m g L
```

---

## 方法2：ロードセル

推奨構成：

```text
サーボ
  ↓
アーム
  ↓
ロードセル
  ↓
固定治具
```

---

## 推奨取得項目

| 項目 | 内容 |
|---|---|
| load_force | 荷重 |
| estimated_torque | 推定トルク |
| constraint_state | 拘束状態 |

---

# 6. 温度観測

## 目的

```text
熱による特性変化を観測する
```

---

## 推奨

```text
ICS内部温度
+
外部温度センサ
```

---

## 理由

ICS内部温度は：

- フィルタ済み
- 遅い
- 内部位置依存

の可能性がある。

---

# 7. 同期

## 最重要項目

すべてのデータに：

```text
共通タイムスタンプ
```

を付与する。

---

## 理想

```text
全観測を単一クロックで収録
```

---

## 最低限

```text
送信時刻
受信時刻
観測時刻
```

を明示する。

# ICS系機能の再現可能性と観測方法

| 機能 | sysid再現 | 主観測手段 | 推奨センサ/治具 |
|---|---|---|---|
| スピード | 可能 | 外部速度観測 | エンコーダ |
| ストレッチ | 可能 | 負荷＋角度誤差 | 重り、ロードセル |
| パンチ | 条件付き可能 | 初動電流・加速度 | 電流計＋エンコーダ |
| デッドバンド | 可能 | 微小角度応答 | 高分解能エンコーダ |
| プロテクション | 可能 | stall監視 | 固定治具＋電流監視 |
| リミッタ | 可能 | 停止位置観測 | エンコーダ |
| 温度制限 | 条件付き可能 | 温度上昇観測 | 温度センサ |
| 電流制限 | 可能 | 電流観測 | 電流計＋負荷 |
| リバース | 可能 | 符号反転確認 | エンコーダ |
| 回転モード | 可能 | 実速度観測 | エンコーダ |
| レスポンス | 可能 | ステップ応答 | エンコーダ |
| ダンピング | 可能 | 減衰観測 | エンコーダ＋負荷 |
| ユーザーオフセット | 可能 | ゼロ点比較 | エンコーダ |

# 機能別の具体的実験

# スピード

## 目的

```text
速度設定値ごとの最大速度、加減速、到達時間を求める
```

---

## 方法

```text
1. 無負荷状態にする
2. speed設定を変更する
3. ステップ入力を与える
4. 外部エンコーダで高速記録する
```

---

## 取得データ

```text
target_angle
external_angle
external_velocity
ics_angle
speed_setting
voltage
current
```

---

# ストレッチ

## 目的

```text
位置保持剛性を求める
```

---

## 方法

```text
1. 一定角度保持
2. アーム先端へ荷重
3. stretch設定変更
4. 角度ズレ量を観測
```

---

## sysid対象

```text
kp
holding_torque_gain
compliance
```

---

# パンチ

## 目的

```text
初動トルク補正を求める
```

---

## 方法

```text
1. 軽い負荷をかける
2. 小さい目標変更を与える
3. punch設定変更
4. 初動加速度・電流ピーク観測
```

---

## 必須観測

```text
外部エンコーダ
+
高速電流観測
```

---

# デッドバンド

## 目的

```text
不感帯幅を求める
```

---

## 方法

```text
±0.1°
±0.2°
±0.5°
...
```

のような微小指示を与え、

```text
実際に反応する閾値
```

を観測する。

---

# プロテクション

## 目的

```text
stall保護条件を求める
```

---

## 方法

```text
1. サーボを拘束
2. 目標角をずらす
3. 電流・時間・脱力タイミング観測
```

---

## 注意

```text
破損リスクがあるため、
短時間・低負荷から始める。
```

---

# 温度制限

## 目的

```text
温度依存特性を求める
```

---

## 方法

```text
1. 一定負荷を長時間与える
2. 温度・電流・応答低下観測
```

---

# 電流制限

## 目的

```text
電流制限と実トルクの関係を求める
```

---

## 方法

```text
1. 負荷を増加
2. current limit設定変更
3. 保持限界観測
```

---

# レスポンス

## 目的

```text
制御応答速度を求める
```

---

## 方法

```text
1. ステップ入力
2. response設定変更
3. 応答波形観測
```

---

# ダンピング

## 目的

```text
速度依存減衰を求める
```

---

## 方法

```text
1. 外部から動かす
2. 減速挙動観測
```

# まとめ

実機sysidでは、

```text
入力
↓
内部状態
↓
実物理挙動
```

の関係を推定する。

そのため、

```text
「何を指示したか」
```

は観測対象そのものであり、

- 指示内容
- 指示時刻
- 指示周期
- 指示パケット

まで記録することが望ましい。

また、

```text
ICS内部値
```

だけでは不十分であり、

- 外部角度
- 外部電流
- 外部負荷

を独立に観測することで、より高精度なsysidが可能になる。
