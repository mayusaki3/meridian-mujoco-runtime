# SILとHILを実現するには

MuJoCoのシミュレーション環境で、PC上で制御ソフトの検証や、実マイコンを使った検証ができると、ロボットの実機が無くても色々テストできてうれしい。SIL (Software In Loop) や HIL (Hardware In Loop) を実現したい。

## 何が必要か？

[sysid の使い方イメージ](./03_システム同定(sysid)をどう扱うか.md) で、制御ドライバと計測プローブを Runtime が呼び出し sysid を実行するという考えを書いた。  
SIL や HIL を実行するには、シミュレーションの結果を制御元に返す仕組みがあれば実現できる。  
そこで、ILコネクタを用意する。

## ILコネクタ (In Loop Connector) の概要

ILコネクタは、制御ドライバと Runtime の間に入るデータ連携及び同期の仕組みとして機能する。  
制御ドライバによって、既定のILコネクタが選択されるイメージ。  
制御ドライバの代わりに、PC上の制御アプリや実マイコンが入る形になる。

## ILコネクタに必要な機能

### sysid用の場合

制御ドライバからデータを受け取り、Runtime に中継する。  
このケースでは実時間モードを想定している。

### SIL用の場合

PC上で制御ソフトからデータを受け取り、Runtime に中継する。  
MuJoCoの1step終了後のデータを受け取り、制御ソフトに中継する。
時間同期は、シミュレーション時間モードを想定している。

### HIL用の場合

実マイコンからデータを受け取り(PlatformIO もしくは独自仕様で連携)、Runtime に中継する。  
MuJoCoの1step終了後のデータを受け取り、実マイコンに中継する。
時間同期は、シミュレーション時間モードを想定している。
そのため実マイコン側は MuJoCo側が1step終了したら1step進むような仕組みを持つ必要がある。

この仕組みでsysidを行うこともでき、その場合は実時間モードで動作させる。
sysidなので計測に特化し、シミュレーション結果の中継はできない。

## シミュレーション結果による信号合成

sysidで使用する計測プローブは、実環境での実測のための物で、シミュレーションでは計算結果を使用する。  
次ステップ計算関数次第で存在しないセンサーの信号を作成できる。  
例えば、ロボットのZMPを計算し、足裏センサーのセンサ値を計算、それをILコネクタ経由で実マイコンに中継するなどが実現できる。

## その他の応用

ILコネクタにより、外部の指令を取り込んだり、シミュレーション結果を送ることができる。
ニソコンVRでコントローラ入力を受け取り、シミュレーション結果を送信、なども対応できる、かもしれない。
