# sysid用 標準機器構成案

## 目的

sysidで以下を安定して収集するための標準機器構成を定義する。

```text
1. 指令ログ
2. 内部状態ログ
3. 外部角度
4. 外部速度
5. 外部電流・電圧
6. 外部負荷
7. 温度
8. 同期タイムスタンプ
```

---

# 1. 標準構成の考え方

## 基本方針

```text
制御指令は必ず記録する
内部値は補助観測として記録する
外部センサー値を正観測として扱う
すべて共通時刻で同期する
```

---

# 2. 標準機器構成

```text
PC
  ├─ sysid制御・ログ収集ソフト
  ├─ MuJoCo / 最適化処理
  │
  ├─ 通信アダプタ
  │    ├─ UART / TTL
  │    ├─ RS485
  │    ├─ CAN
  │    └─ PWM出力
  │
  ├─ センサーロガーMCU
  │    ├─ 外部エンコーダ
  │    ├─ 電流・電圧センサー
  │    ├─ ロードセル
  │    └─ 温度センサー
  │
  └─ 電源・安全系
       ├─ 安定化電源
       ├─ 電流制限
       ├─ 非常停止
       └─ ヒューズ
```

# 6. 機器別役割

## 6.1 PC

## 役割

```text
試験シナリオ実行
指令生成
ログ統合
sysid処理
結果保存
MuJoCo検証
```

## 必須機能

```text
高精度時刻管理
CSV / JSONL / Parquet保存
生データ保存
正規化データ保存
```

---

## 6.2 センサーロガーMCU

## 役割

```text
外部センサーを高速・安定周期で読む
共通タイムスタンプを付ける
PCへストリーミングする
```

## 推奨

```text
1kHz以上の収集周期
複数ADC入力
SPI / I2C / UART対応
割り込み入力対応
```

---

## 6.5 電流・電圧センサー

## 目的

```text

```


```text

```

# 8. 同期要件

## 必須

```text
すべてのログに timestamp を付ける
指令送信時刻と応答受信時刻を分ける
外部センサー取得時刻を分ける
PC受信時刻も保存する
```

## 推奨

```text
MCU側単調増加カウンタ
PC側時刻
step_index
sequence_id
```

| 対象 | 推奨 |
|---|---:|
| 指令ログ | 指令ごと |
| 外部角度 | 500Hz〜2kHz |
| 外部電流 | 500Hz〜5kHz |
| 外部電圧 | 500Hz〜5kHz |

| 温度 | 1Hz〜10Hz |

# 15. まとめ

標準機器構成は、最初から以下の3系統を分けて設計するのがよい。

```text
1. 指令・内部状態取得系
2. 外部センサー観測系
3. 負荷・安全・治具系
```

最小構成では、

```text
外部角度
外部電流
指令ログ
既知負荷
```

を取れるようにする。

推奨構成では、

```text
ロードセル
温度
生通信ログ
高精度同期
```

を追加する。

HILや移動ロボットまで考える場合は、

```text
step同期
IMU
ロボット姿勢
床面摩擦
スリップ
```

も標準機器構成に含められるよう、初期段階から拡張可能な構成にしておく。
