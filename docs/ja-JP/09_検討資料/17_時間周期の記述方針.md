# 時間周期の記述方針

## 背景

本プロジェクトでは、以下のように複数の時間概念が存在する。

```text
sysid のサンプリング周期
MuJoCo timestep
制御周期
HIL step 同期
通信周期
センサー収集周期
ログ周期
```

そのため、

```text
Hz（周波数）
時間間隔（s / ms / μs）
```

の使い分けを整理する必要がある。

---

# 基本方針

## 要件定義・検討段階

```text
周波数主体
+
時間間隔併記
```

とする。

例：

```text
1 kHz（1 ms間隔）
500 Hz（2 ms間隔）
5 kHz（0.2 ms間隔）
```

---

## 仕様・実装段階

```text
時間間隔主体
```

とする。

例：

```text
制御周期:
  1 ms

MuJoCo timestep:
  0.5 ms

通信周期:
  2 ms
```

必要に応じて補足として、

```text
（1 kHz相当）
```

を付与してよい。

---

# 理由

## 要件定義・検討段階

要件定義では、

```text
十分細かいか
粗すぎないか
必要帯域を満たしているか
```

を人間が議論するため、

```text
Hz表記
```

の方が直感的である。

例：

```text
50 Hz（20 ms間隔）
```

を見ると、

```text
高速サーボ観測には粗い
```

と判断しやすい。

---

## 仕様・実装段階

実装では、

```text
scheduler
timestamp
timeout
sleep
step
Δt
```

など、

```text
時間間隔
```

で処理される。

また、MuJoCo 自体も、

```text
model.opt.timestep
```

のように、

```text
時間間隔ベース
```

で動作する。

そのため、仕様・実装では、

```text
ms / s / μs
```

主体とする。

---

# 用途別推奨表記

| 用途 | 推奨表記 |
|---|---|
| センサー観測 | `1 kHz（1 ms間隔）` |
| ログ収集 | `1 kHz（1 ms間隔）` |
| サンプリング能力 | `5 kHz（0.2 ms間隔）` |
| 制御周期 | `1 ms` |
| MuJoCo timestep | `0.5 ms` |
| 通信周期 | `2 ms` |
| timeout | `10 ms` |
| PWMパルス幅 | `1500 μs` |
| Runtime内部処理 | `s` |
| Dataset metadata | 周波数・間隔両方保持 |

---

# sysid における推奨

## 検討資料

```text
角度収集:
  1 kHz（1 ms間隔）

電流収集:
  5 kHz（0.2 ms間隔）
```

のように併記する。

---

## 実装仕様

```text
angle_sampling_interval_s:
  0.001

current_sampling_interval_s:
  0.0002
```

のように、

```text
時間間隔ベース
```

で定義する。

---

# HIL における推奨

HILでは、

```text
step token
wait
timeout
同期
```

が中心となるため、

```text
時間間隔主体
```

とする。

例：

```text
control_period_ms:
  1

step_timeout_ms:
  10
```

---

# Dataset metadata 推奨

Dataset では、

```text
人間可読性
+
内部計算容易性
```

の両方を満たすため、

```text
周波数
時間間隔
```

を両方保持する。

例：

```json
{
  "sampling": {
    "rate_hz": 1000,
    "interval_s": 0.001
  }
}
```

---

# ドキュメント記述ルール

## 初出

初出では、

```text
周波数 + 時間間隔
```

を併記する。

例：

```text
角度センサーは 1 kHz（1 ms間隔）で収集する。
```

---

## 以後

同一文書内では、文脈に応じて片方を省略してよい。

例：

```text
以後、本資料では「1 kHz収集」と表記する。
```

---

# まとめ

本プロジェクトでは、

```text
要件定義・検討:
  周波数主体 + 時間間隔併記

仕様・実装:
  時間間隔主体
```

を基本方針とする。

これにより、

```text
人間の理解しやすさ
MuJoCo timestepとの整合
HIL step同期との整合
runtime実装容易性
```

を両立する。
