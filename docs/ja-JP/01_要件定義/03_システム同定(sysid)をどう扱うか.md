# システム同定(sysid)をどう扱うか

MuJoCo上のシミュレーション精度を向上するため、システム同定(sysid)機能を提供したい。  

本システムでは、利用者が手持ちのアクチェーターやロボットの実測データを元に sysid を実行し、その結果を MuJoCoデータセットに保存することで、シミュレーションへ反映できるようにする。

## sysid の対応例

- アクチェーター特性
  - PID応答
  - トルク特性
  - 速度特性
  - 制御遅延
  - デッドゾーン
  - 個体差

- ロボット組み立て状態
  - 関節摩擦
  - 重量バランス
  - 慣性変化
  - 配線や機構による負荷
  - 組み立て精度差

- シミュレーション環境
  - 接地摩擦
  - 床材特性
  - 外乱条件

## sysid の使い方イメージ

本システムでは、以下のような使い方を想定している。

### 1. sysid対象の初期定義  

MJCFを使用し、対象の種類や必要なパラメータ定義（角度や電圧、計算式内の変数など）を行う。  
ここで定義したパラメータ（customも含む）がsysidの結果を更新する対象になる。  
必要なら、メッシュなども作成する。  
MuJoCo標準ではない計算が必要な場合は、custom関数呼び出しを定義できる。
custom関数呼び出しをMJCFで定義した場合、該当する custom関数が無ければ作成が必要。
custom関数には、MuJoCo上での次ステップ計算関数と、sysid時の同定評価関数が含まれる。

### 2. 計測対象と制御方法を決定  

既存の計測プローブがあれば選択、なければ新規に計測プローブを作成する。
- 計測対象は、計測プローブが対応しているMJCFに定義したパラメータ（角度や電圧など）。  
- 計測プローブが対応するMJCFと、定義されたどのパラメータに対応するかは、計測プローブの開発者が自由に決めてよい。

既存の制御ドライバがあれば選択、なければ新規に制御ドライバを作成する。 
- 制御方法は、PWMやシリアルコマンドによるモーション再生、電圧制御などで、MJCFでパラメータ定義されたものとなる。 
- 制御ドライバ次第で、CSV入力でも実マイコンからの入力でも対応できる。
- 制御ドライバが対応する入力や制御対象は、制御ドライバの開発者が自由に決めてよい。

計測プローブ、制御ドライバは、Runtime から呼び出す想定。
どちらも Runtime に対して、どの MJCF の どのパラメータにアクセスするかを公開する必要がある。

### 3. 実対象を制御し、計測を行う  

計測開始で、Runtime は MJCF で定義されたパラメータセットを作成し、制御ドライバが終了するまで制御ドライバが設定したデータと計測プローブにより収集されたデータを更新しながら結果を記録する。  
観測周期は 0.1 ms間隔 (10kHz) が理想。そのため実マイコンで計測・バッファリングし、USBやWiFiでPCに転送する必要があるかもしれない。

### 4. シミュレーション対象を制御し、sysidを行う  

sysid開始で、Runtime は、収集されたデータに含まれる制御ドライバによる指令値で MuJoCo のパラメータセットを更新し、シミュレーションを行う。
指令値がなくなるまで Runtime はその結果を収集し保存する。

3.で収集された結果と、4.で収集された結果の誤差が少なくなるよう、計算式内の変数パラメータ調整を繰り返す。

一定の誤差内に収まったとき、MJCF内の計算式内の変数パラメータを更新し、sysyidが終了する。
custom関数内の処理を変更し、再実行のケースも想定したい。
