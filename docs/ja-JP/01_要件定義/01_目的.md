# meridian-mujoco-runtime の目的

## 概要

meridian-mujoco-runtime は、MuJoCo を中心としたロボット実行基盤（Runtime Platform）である。

単なるシミュレーターではなく、

- ロボット制御
- 物理シミュレーション
- 実機接続
- HIL / SIL
- VR連携
- AI制御
- Meridian 系通信
- センサー・アクチュエーター統合

を一つの実行環境として接続し、同じ構成・同じデータ構造のまま、シミュレーションから実機へ段階的に移行できることを目的とする。

本システムは、「シミュレーション専用環境」と「実機専用環境」を分離するのではなく、それらを連続した Runtime として扱う。

これにより、

- シミュレーションで検証した制御を実機へ移行する
- 実機で取得した特性をシミュレーションへ反映する
- VR空間へ物理結果を同期する
- AIによる制御実験を安全に行う
- 複数ロボットや複数環境を共通基盤で扱う

といったことを可能にする。

---

## 解決したい課題

現在のロボット開発では、

- シミュレーター
- 実機制御
- VR
- 通信
- ログ解析
- AI制御

が個別に分離されていることが多い。

その結果、

- シミュレーションでは動くが実機では動かない
- 実機調整内容がシミュレーションへ戻らない
- VRとの同期構造が毎回異なる
- ロボットごとに制御構造を作り直す
- ハードウェア変更時の影響範囲が大きい

といった問題が発生しやすい。

meridian-mujoco-runtime は、これらを統合された Runtime として扱うことで、ロボット開発全体を継続的かつ再利用可能な形へ整理することを目指す。

---

## meridian-mujoco-runtime が目指すもの

meridian-mujoco-runtime は、以下を実現することを目的とする。

- MuJoCo 上での高精度ロボットシミュレーション
- 実機特性を反映したアクチュエーター再現
- 実機とシミュレーションを接続した HIL / SIL
- Meridian 系通信による制御統合
- VR空間とのリアルタイム同期
- AI・自律制御実験の実行基盤化
- ロボット構成要素の再利用
- シミュレーションから実機への段階的移行
- ロボット競技・協調・遠隔操作への応用

本プロジェクトは、「ロボットを動かす」だけではなく、

「現実・仮想・AI・人間操作を統合した Runtime を作る」

ことを最終目標とする。

---

## 想定する利用シーン

本 Runtime は、以下のような用途を想定する。

- ヒューマノイドロボット開発
- ホビーロボット開発
- Meridian 対応ロボット制御
- VR連携ロボット
- AI制御実験
- 遠隔操作
- ロボット競技
- 教育用途
- 制御研究
- アクチュエーター特性研究
- sysid（System Identification）
- デジタルツイン
- 実機移行前検証

---

## 最終目標

最終的には、

- 実機
- MuJoCo
- VR
- AI
- 制御基板
- センサー
- 通信
- ログ解析

を一つの Runtime として扱える世界を目指す。

そして、

「同じロボットが、シミュレーションでも、VRでも、実機でも、同じように存在し、同じように動く」

状態を実現する。

---

## スコープ

本プロジェクトは Runtime 基盤を対象とする。

個別ロボット固有の機構設計や、特定アプリケーション専用ロジックは対象外とする。

また、描画エンジン・ゲームエンジン・AIモデルそのものの実装を主目的とはせず、それらを接続・実行するための Runtime 基盤の構築を主目的とする。
