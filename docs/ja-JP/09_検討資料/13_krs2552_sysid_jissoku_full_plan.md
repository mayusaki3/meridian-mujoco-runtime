## 4.3 器具選びの注意

### 外部角度センサー

おすすめは磁気式絶対角エンコーダ。

```text
最低：AS5600系
推奨：AS5048A系
```

取り付けは、サーボホーン中心に磁石、正面にセンサー基板。

```text
サーボ出力軸
  ↓
サーボホーン中心の磁石
  ↓
磁気エンコーダ基板
```

### 電流・電圧センサー

```text
INA219 / INA226：入門用。平均的な電流を見るには便利。
シャント抵抗 + 高速ADC：初動電流やパンチを見るなら推奨。
```

### ロードセル

```text
HX711：安価で扱いやすいが、動的応答には遅い。保持力試験向き。
高速ロードセルアンプ：重り付きステップ応答まで見たい場合に有利。
```

### 固定治具

3Dプリントだけだとたわみやすい。最初から金属固定がよい。

```text
アルミフレーム
L金具
金属サーボブラケット
万力
厚めの木板 + 金属金具
```

### アーム

```text
アルミ平板 2〜3mm厚
長さ 50mm / 100mm の穴を開ける
穴位置をノギスで測る
```

# 5. 共通セットアップ

## 5.1 サーボ固定

```text
1. KRS-2552を金属L金具やアルミフレームに固定する
2. サーボホーンに短いアームを付ける
3. アームには50mm / 100mm位置に穴を開ける
4. 可動範囲は最初±45°程度に制限する
5. 物理ストッパを付ける
6. 指やケーブルが巻き込まれないようにする
```

---

## 5.2 外部角度センサー取り付け

```text
1. サーボホーン中心に磁石を貼る
2. 磁石の正面に角度センサー基板を固定する
3. センサー基板がアームと一緒に動かないようにする
4. ケーブルをアームに当てない
5. 0°位置を校正する
```

注意。

```text
磁石中心とセンサー中心をなるべく合わせる
センサー基板のガタをなくす
角度が飛ぶ場合は磁石位置を見直す
```

---

## 5.3 電流・電圧センサー接続

```text
安定化電源 +
  ↓
電流・電圧センサー
  ↓
KRS-2552
  ↓
安定化電源 -
```

ログする値。

```text
supply_voltage_v
supply_current_a
```

---

## 5.4 サンプリング周波数の目安

```text
外部角度：500Hz〜2kHz
外部電流：500Hz〜5kHz
外部電圧：500Hz〜5kHz
ロードセル：100Hz〜1kHz
温度：1Hz〜10Hz
ICS内部値：取れる最大周期
```

最初は1kHzを目標にすると扱いやすい。

# 6. A〜F 各メニューの具体的実測方法

---

## A. ゼロ点確認

### 目的

```text
指令角と外部角度の0°を合わせる
回転方向のプラス・マイナスを確認する
```

### 使う器具

```text
KRS-2552
PC + Dual USBアダプターHS
外部角度センサー
安定化電源
サーボ固定治具
```

### 手順

```text
1. サーボを固定する
2. 外部角度センサーを取り付ける
3. 電源を入れる
4. 0°を指令する
5. 2〜3秒待つ
6. 外部角度の平均値を記録する
7. その値を zero_offset とする
8. +10°を指令する
9. 外部角度が増えるか確認する
10. -10°を指令する
11. 外部角度が減るか確認する
12. 3回繰り返す
```

### 記録するもの

```text
target_position_rad
external_angle_rad
zero_offset_rad
sign
voltage_v
current_a
```

### OK判定

```text
0°指令時の外部角度ばらつきが小さい
+10°で外部角度が増える
-10°で外部角度が減る
```

逆なら `sign = -1` として保存する。

## B. 無負荷ステップ応答

### 目的

```text
速さ
遅れ
最大速度
止まり方
オーバーシュート
整定時間
```

を見る。

### 使う器具

```text
KRS-2552
外部角度センサー
電流・電圧センサー
安定化電源
サーボ固定治具
軽いアーム
MCUロガー
```

### 手順

```text
1. 重りは付けない
2. アームだけ付ける
3. ログを開始する
4. 0°で2秒保持する
5. +10°へ指令する
6. 2秒待つ
7. 0°へ戻す
8. 2秒待つ
9. -10°へ指令する
10. 2秒待つ
11. 0°へ戻す
12. これを5回繰り返す
```

次に角度を増やして同じことをする。

```text
±10°
±20°
±30°
±45°
```

### 取る値

```text
指令時刻
目標角
target_position_rad
external_angle_rad
external_velocity_rad_s
supply_current_a
supply_voltage_v
temperature_c
```

### 見るポイント

```text
指令から動き始めるまでの時間
最大速度
目標角への到達時間
オーバーシュート
整定時間
電流ピーク
```

### コツ

```text
最初は±10°だけで動作確認する
問題なければ±30°、±45°へ広げる
毎回同じ待ち時間にする
ログ開始前に1秒以上静止させる
温度が上がりすぎたら休ませる
```

## C. 微小指令・デッドバンド・初動応答

### 目的

```text
どれくらい小さい指令だと動かないか
動き出しにどれくらい遅れるか
初動電流がどう出るか
```

を見る。

### 使う器具

```text
高分解能外部角度センサー
電流・電圧センサー
安定化電源
サーボ固定治具
MCUロガー
```

### 手順：デッドバンド

```text
1. 0°へ指令する
2. 2秒待つ
3. +0.1°へ指令する
4. 1秒待つ
5. 外部角度が動いたか見る
6. 0°へ戻す
7. +0.2°で同じことをする
8. +0.5°で同じことをする
9. +1.0°で同じことをする
10. +2.0°で同じことをする
11. マイナス方向も同じように測る
```

測る指令例。

```text
+0.1°
+0.2°
+0.5°
+1.0°
+2.0°
-0.1°
-0.2°
-0.5°
-1.0°
-2.0°
```

### 手順：初動応答

```text
1. 0°で2秒静止する
2. +1°へ指令する
3. 外部角度と電流を記録する
4. 0°へ戻す
5. +2°で同じことをする
6. +5°で同じことをする
7. マイナス方向も測る
```

### 動いた判定

```text
外部角度がセンサーノイズの3倍以上変化
または
0.05〜0.1°以上の変化が50ms以上続く
```

### 見るポイント

```text
動き始める最小指令角
初動遅れ
初動電流ピーク
小指令で動く/動かない判定
```

### コツ

```text
過度にフィルタしない
毎回0°へ戻してから測る
反転方向の影響を避けたい時は、毎回同じ方向から0°へ入る
```

## D. 方向反転・バックラッシュ

### 目的

```text
逆方向へ指令した時に、どれくらい遊びがあるか
反転してから実軸が動くまで何msか
```

を見る。

KRS-2552のようなギア付きサーボでは、ここがMuJoCo再現にかなり効く。

### 使う器具

```text
外部角度センサー
サーボ固定治具
軽いアーム
必要なら小さい重り
電流・電圧センサー
```

### 方法1：通電したまま指令反転で測る

こちらを主データにする。

```text
1. -30°へ指令する
2. 2秒保持する
3. +30°へ指令する
4. 2秒保持する
5. -30°へ指令する
6. 2秒保持する
7. これを5回繰り返す
```

小さい角度でもやる。

```text
-10° → +10°
+10° → -10°
-30° → +30°
+30° → -30°
```

### 方法2：トルクOFFで手動確認

これは補助用。

```text
1. サーボを脱力する
2. アームを指でそっと左右へ動かす
3. 外部角度センサーで遊び幅を読む
4. 3回繰り返す
```

### 見るポイント

```text
反転指令時刻
実軸が動き始めた時刻
反転後に失われた角度
左右差
```

### 注意

```text
手動バックラッシュは参考値
通電中の反転挙動とは別物
sysidの主データには方法1を使う
```

## E. 重り付きステップ応答

### 目的

```text
負荷がある時の遅れ
速度低下
沈み込み
電流増加
トルク不足
```

を見る。

### 使う器具

```text
サーボ固定治具
アルミ製アーム
重り
外部角度センサー
電流・電圧センサー
安定化電源
温度センサー
必要ならロードセル
```

### 方法1：アームに重りを吊るす

一番簡単。

```text
1. アーム長 L を測る
2. 例：L = 50mm
3. アーム先端に穴を開ける
4. 糸で重りを吊るす
5. アームが水平付近になる角度で測る
```

水平付近なら簡易トルクはこれでよい。

```text
τ = m g L
```

角度変化まで考えるなら、アーム角を水平基準で theta として以下。

```text
τ(t) = m g L cos(theta)
```

例。

```text
m = 0.05 kg
L = 0.05 m

tau = 0.05 * 9.80665 * 0.05
    = 約0.0245 Nm
```

最初の重り例。

```text
50g
100g
200g
```

公称最大トルク付近は攻めない。家庭測定では破損リスクが高い。

### 方法2：滑車で重りを吊るす

少し良い方法。

```text
1. アーム先端に糸を付ける
2. 小さい滑車を通す
3. 重りを下に吊るす
4. アームに一定方向の負荷をかける
```

利点。

```text
重りを交換しやすい
角度が変わっても負荷方向を作りやすい
```

欠点。

```text
滑車の摩擦が入る
治具が少し面倒
```

### 方法3：ロードセルで測る

推奨構成。

```text
1. アーム先端をロードセルへ接続する
2. ロードセルを固定台に取り付ける
3. アームが押す/引く力を測る
4. tau = F * L でトルクに変換する
```

ロードセルは静的な保持力測定に向いている。動的なステップ応答まで正確に見たい場合は、ロードセルアンプの速度にも注意する。

### 重り付きステップの手順

```text
1. 無負荷ステップが正常に取れていることを確認する
2. 50gの重りを付ける
3. 0°で2秒保持する
4. +10°へ指令する
5. 2秒待つ
6. 0°へ戻す
7. -10°へ指令する
8. 2秒待つ
9. 5回繰り返す
10. 問題なければ100g、200gで繰り返す
```

負荷あり試験では、まず大角度を避ける。

```text
推奨：±10°、±20°
慣れてから：±30°
最初は避ける：±45°以上 + 重り
```

### 注意

```text
重りを急に落とさない
アームが暴れないようにする
温度を記録する
電流が急増したら停止する
異音が出たら中止する
```

## F. 保持力・沈み込み

### 目的

```text
止まって支える力
stretch相当の保持剛性
負荷による角度沈み込み
保持中の電流
```

を見る。

### 使う器具

```text
サーボ固定治具
アーム
重り
外部角度センサー
電流・電圧センサー
温度センサー
可能ならロードセル
```

### 方法1：重りを段階的に付ける

```text
1. アームを水平に近い角度で保持する
2. 重りなしで3秒ログする
3. 50gを付ける
4. 5秒ログする
5. 重りを外す
6. 0°へ戻す
7. 100gで同じことをする
8. 200gで同じことをする
```

### 方法2：小さい容器に重りを足す

衝撃を減らす方法。

```text
1. アーム先端に小さい袋や容器を吊るす
2. ワッシャーやナットを少しずつ入れる
3. 各重量で5秒ログする
```

利点。

```text
負荷をなめらかに増やせる
サーボへの衝撃が少ない
```

### 方法3：ロードセル + ネジで押す

研究室評価に近い方法。

```text
1. アーム先端にロードセルを当てる
2. ネジ機構で少しずつ押す
3. 力 F を読む
4. 外部角度の沈み込みを読む
5. tau = F * L でトルク化する
```

### 計算するもの

```text
沈み込み角度 = 重りあり角度 - 重りなし角度

保持剛性 ≒ 負荷トルク / 沈み込み角度
```

角度はradで計算する。

### 記録するもの

```text
target_position_rad
external_angle_rad
angle_drop_rad
mass_kg
arm_length_m
estimated_torque_nm
supply_current_a
supply_voltage_v
temperature_c
```

# 11. MuJoCo側のモデル化方針

単純なMJCFのposition actuatorだけでKRS-2552再現と言い切らない。

推奨は以下。

```text
target_position
↓
command_delay
↓
target_filter / response
↓
deadband
↓
backlash / hysteresis
↓
velocity_limit
↓
PD-like torque model
↓
static + viscous friction
↓
torque_limit
↓
MuJoCo joint torque
```

つまり、外部のActuatorBehaviorModelでトルクを計算し、MuJoCoへ入れる。

いきなり全部同時に最適化しない。

```text
1. 無負荷ステップだけで合わせる
   - command_delay_s
   - control_time_constant_s
   - velocity_limit_rad_s
   - kp
   - kd
   - viscous_friction

2. 重り付きデータを追加する
   - torque_limit_nm
   - holding_stiffness
   - current_to_torque_gain

3. 微小指令を追加する
   - deadband_rad
   - static_friction_nm
   - minimum_torque

4. 方向反転データを追加する
   - backlash_rad
   - hysteresis

5. 未使用データで検証する
```

# 14. 最重要まとめ

人間がやることは、最終的にはこれ。

```text
1. 条件を固定する
2. 指令を記録する
3. 外部角度を測る
4. 電流・電圧を測る
5. 重りで負荷を作る
6. 同じ試験を5回繰り返す
7. CSVで保存する
8. MuJoCoと未使用検証データで比較する
```

一番大事なのはこれ。

```text
外部角度を正解データとして取る。
ICS内部値だけで判断しない。
```
